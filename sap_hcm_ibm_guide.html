Basic Question 1
What is SAP HCM and why do companies use it?

SAP HCM (Human Capital Management) is a core module within SAP ERP that manages the complete employee lifecycle — from hiring through to retirement. It is also referred to as SAP HR.

Why companies use it
Single source of truth
Eliminates data silos between HR, Finance, and Operations
Legal compliance
Country-specific payroll rules, tax calculations, statutory deductions
Full lifecycle coverage

Core Question 2
Explain the difference between Personnel Administration and Organizational Management.
Personnel Administration (PA)

Focuses on the individual employee — their personal, financial, and contractual data.

Data is stored in structured Infotypes with date-range validity.

Answers: "Who is this person?"

PA30
PA20
PA40
Organizational Management (OM)

Focuses on the company structure — org units, jobs, positions, and cost centers.

Built using object-relationship model: O → S → C → P linkages.

Answers: "Where does this person sit?"

PPOME
PP01

OOrg UnitDepartment or business unit (e.g., Finance Dept)

Integration point: When you assign a Position (S) to an employee in PA, SAP automatically inherits org data from OM into Infotype 0001 (Org Assignment) — this is the PA ↔ OM integration switch.

Scenario Question 3
A company hires a new employee. Explain step-by-step how you handle this in SAP.
Step-by-step hiring process
  • 1
    Verify Position in OM — PPOME

    Confirm the Position (S) exists under the correct Org Unit with the right Job and Cost Center. Create it if missing.

  • 2
    Initiate Personnel Action — PA40

    Select action type Hiring. Enter Entry Date, Company Code, Personnel Area, Employee Group/Subgroup.

  • 3
    Maintain Infotypes (system-prompted sequence)

    SAP auto-prompts each required Infotype in the Personnel Action sequence:

InfotypeNameData Entered
0000ActionsAction type = Hiring, Entry date
0001Org AssignmentPosition, Org Unit, Cost Center
0002Personal DataName, DOB, Gender, Nationality
0006AddressesResidential address
0007Planned Working TimeWork schedule assignment
0008Basic PayWage type, salary amount
0009Bank DetailsAccount number for salary payment
  • 4
    Time Management Setup

    Assign Work Schedule Rule via IT0007. Configure leave entitlements and absence quotas.

  • 5
    Verify & Audit — PA20

    Display all maintained Infotypes. Run RPUAUD00 audit report if required. Employee is now active and payroll-ready.

Advanced Question 4
What are Infotypes? Give examples and explain their role.

Infotypes are structured data containers that store HR information for an employee. Each Infotype acts as a dedicated form/table for a specific category of data, with built-in date-range validity enabling full historical tracking.

Structure of Infotype 0008 — Basic Pay

PERNR : 10001 (Personnel Number)
BEGDA : 01.04.2026 (Valid From)
ENDDA : 31.12.9999 (Valid To — open-ended)
Wage : M001 → ₹50,000 (Wage Type + Amount)
Table : PA0008 (Database table)

Key Infotypes to know
InfotypeNamePurpose
0000ActionsRecords all HR actions — hire, terminate, transfer, promote
0001Org AssignmentLinks employee to Position, Org Unit, Cost Center
0002Personal DataName, DOB, marital status, nationality
0006AddressesHome/mailing/emergency address
0007Planned Working TimeShift assignment, work schedule rule
0008Basic PaySalary structure, wage types, pay scale
0009Bank DetailsBank account number for salary disbursement
0014Recurring Payments/Ded.Monthly allowances and deductions
0015Additional PaymentsOne-time bonuses, arrears
0019Monitoring of TasksProbation dates, contract expiry reminders

2002AttendancesOvertime and special attendance records

Interview tip: Infotypes 0000–0999 are SAP standard. Infotypes 9000+ are customer-defined (custom). Multiple records of the same Infotype with different date ranges store complete salary/action history — this is called time-based data management.

  • 1
    Release Payroll

    Lock the period to prevent further master data changes during the run.

  • 2
    Start Payroll — PC00_M40_CALC (India)

    Engine reads PA and TM data, applies wage types, calculates net pay.

  • 3
    Review Payroll Results — PC00_M99_CWTR

    Verify gross-to-net for each employee. Check error log for exceptions.

  • 4
    Post to Accounting — PCP0

    Generate FI/CO posting documents. Salary expense hits cost centers.

  • 5
    Bank Transfer & Payslip

    Generate DME file for NEFT. Distribute payslips via PC00_M40_CEDT.

Retroactive Payroll: If IT0008 is changed with a backdated effective date, SAP automatically recalculates past payroll periods and adjusts the difference in the current run. This is one of SAP's most powerful payroll capabilities — a key point to highlight in interviews.

Core concepts to remember

Infotype validity: Every Infotype has BEGDA (begin date) and ENDDA (end date). Multiple records with different date ranges store complete history — enabling time-based data management.

PA ↔ OM integration: Assigning a Position to an employee in PA automatically inherits Org Unit, Cost Center, and Job from OM via Infotype 0001.

Retroactive payroll: Backdated changes to IT0008 trigger automatic recalculation of past periods, with differences posted in the current payroll run.

Custom Infotypes: Standard ITs are 0000–0999. Customer-defined ITs start at 9000+. Subtypes within an IT allow further categorization (e.g., IT0006 subtype 1 = permanent, 2 = temporary address).
