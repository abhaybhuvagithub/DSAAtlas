Sorting Algorithms
Merge · Quick · Heap · Counting/Radix
01 Merge Sort
Stable O(n log n) O(n) space
CaseTimeSpace
BestO(n log n)O(n)
AverageO(n log n)
WorstO(n log n)
function mergeSort(arr): if len(arr) <= 1: return arr mid = len(arr) / 2 left = mergeSort(arr[0..mid]) right = mergeSort(arr[mid..]) return merge(left, right)
✅ Strengths
  • Guaranteed O(n log n) in all cases
  • Stable sort
⚠️ Trade-offs
  • O(n) extra space
02 Quick Sort
Unstable O(n log n) avg In-place
CaseTimeSpace
BestO(n log n)O(log n)*
AverageO(n log n)
WorstO(n²)
function quickSort(arr, low, high): if low < high: p = partition(arr, low, high) quickSort(arr, low, p-1) quickSort(arr, p+1, high)
⚠️ Trade-offs
  • Worst-case O(n²) with bad pivots
  • Use randomized / median-of-three pivot selection

Searching Algorithms
Binary · BFS · DFS
01 Binary Search
O(log n) Sorted required
CaseTimeSpace
BestO(1)O(1)
AverageO(log n)
WorstO(log n)
function binarySearch(arr, target): low = 0; high = len(arr) - 1 while low <= high: mid = low + (high - low) / 2 if arr[mid] == target: return mid elif arr[mid] < target: low = mid + 1 else: high = mid - 1 return -1
🐛 Corner Cases
  • Overflow-safe mid formula
  • Duplicates → leftmost/rightmost variants

Memoization · Tabulation
DP Dynamic Programming — Core Concepts
Optimal substructure Overlapping subproblems
Two Approaches
  • Top-down: recursion + memo cache
  • Bottom-up: iterative table (tabulation)
  • Often space-optimizable (2D → 1D)
Common Pitfalls
  • Wrong base cases / off-by-one
  • Incorrect recurrence
  • Not resetting memo between tests
