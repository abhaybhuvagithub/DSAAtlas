Array
Contiguous memory, index-based
Access O(1) Insert O(n) Static / Dynamic
OperationBestAverageWorstSpace

✅ Strengths
  • O(1) random access via index
  • Cache-friendly — excellent spatial locality
  • Supports binary search on sorted arrays
⚠️ Weaknesses / Trade-offs
  • Insertions/deletions in middle are O(n) due to shifting
  • Resizing is expensive O(n) (dynamic arrays)

  • Binary search mid overflow: use low + (high-low)/2
🚀 Applications
  • Lookup tables, buffers, image matrices
  • Backing storage for stacks/queues (array-based)

  • O(1) insert/delete at head (or both ends with tail/doubly)

Circular Linked List
Tail points back to head (no null end)

* O(1) if you already have a pointer to insertion point (or maintain a tail pointer)
✅ Strengths
  • No null end — natural cyclic traversal
  • Great for round-robin iteration
  • O(1) head ops if you keep a tail pointer
⚠️ Weaknesses
  • Traversal termination is tricky (risk infinite loop)
  • Still O(n) search like a normal linked list
🐛 Corner Cases
  • Single node: node.next points to itself
  • Deleting head must update tail.next
  • Empty vs one-element list handling
🚀 Applications
  • Round-robin scheduling
  • Playlist looping
  • Token ring / cyclic iteration patterns

Sorting Algorithms
Merge · Quick · Heap · Counting/Radix
01 Merge Sort
Stable O(n log n) O(n) space
CaseTimeSpace
BestO(n log n)O(n)
AverageO(n log n)
WorstO(n log n)
function mergeSort(arr): if len(arr) <= 1: return arr mid = len(arr) / 2 left = mergeSort(arr[0..mid]) right = mergeSort(arr[mid..]) return merge(left, right)
✅ Strengths
  • Guaranteed O(n log n) worst-case
  • Stable sort
⚠️ Trade-offs
  • Extra O(n) space
02 Quick Sort
Unstable O(n log n) avg In-place
CaseTimeSpace
BestO(n log n)O(log n)*
AverageO(n log n)
WorstO(n²)
function quickSort(arr, low, high): if low < high: p = partition(arr, low, high) quickSort(arr, low, p-1) quickSort(arr, p+1, high)
✅ Strengths
  • Fast in practice for many inputs
  • In-place (recursion stack aside)
⚠️ Trade-offs
  • Worst case O(n²) (use randomized/median pivot)

Dynamic Programming
Memoization · Tabulation
DP Dynamic Programming — Core Concepts
Optimal substructure Overlapping subproblems
Two Approaches
  • Top-down: recursion + memo cache
  • Bottom-up: iterative table (tabulation)
  • Often space-optimizable (2D → 1D)
Common Pitfalls
  • Wrong base cases / off-by-one
  • Incorrect recurrence
  • Not resetting memo between tests
